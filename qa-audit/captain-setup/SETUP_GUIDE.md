# 🎯 دليل تفعيل QAYDAO AI Assistant (Captain)

> **للموظف:** هذا الدليل خطوة بخطوة لتفعيل المساعد الذكي في Chatwoot.
> **الوقت المتوقع:** 30-45 دقيقة
> **المتطلبات:** صلاحيات Administrator على Chatwoot

---

## 📋 ملخص ما ستفعله

```
1. إنشاء Assistant         (5 دقائق)
2. ضبط Personality + Rules (10 دقائق)
3. رفع 6 Documents          (10 دقائق)
4. إنشاء 5 Scenarios        (10 دقائق)
5. ربط Assistant بـ Inbox   (5 دقائق)
6. اختبار من خلال widget    (5 دقائق)
```

---

## 🚀 الخطوة 1: إنشاء Assistant

1. ادخل: **`https://chat.qaydao.com/app/accounts/1/captain/assistants`**
2. اضغط زر **"New Assistant"** في الأعلى يميناً
3. املأ النموذج:

| الحقل | القيمة |
|------|--------|
| **Name** | `QAYDAO AI` |
| **Description** | `المساعد الذكي لخدمة عملاء قيدداو — يرد على الاستفسارات خارج أوقات العمل` |
| **Language** | `Arabic` (إن وُجد، وإلا اختر `English` — لا يؤثر) |

4. اضغط **Create**

---

## ⚙️ الخطوة 2: ضبط Personality + Rules

بعد الإنشاء، ادخل على Assistant وافتح تبويب **"Settings"** أو **"Configuration"**.

### 2.1 Personality / Instructions (الأهم!)

**انسخ هذا النص بالكامل والصقه في حقل "Personality" أو "Instructions":**

```
أنت QAYDAO AI — المساعد الذكي لشركة قيدداو، المتخصصة ببيع الأثاث الفاخر بالتعاون مع مصانع فوشان الصينية. تعمل خارج أوقات الدوام الرسمي.

🎯 مهمتك:
- الرد على استفسارات العملاء بدقة وودّ ومهنية باللهجة السعودية
- الإجابة فقط على ما هو موجود في مصادر المعلومات المرفقة
- التحويل الفوري لموظف بشري عند: الشكاوى، الاسترجاع، البنوك، الغضب، عدم اليقين

🚫 ممنوع تماماً:
- اختراع معلومات غير موجودة في المصادر
- وعد بمواعيد تسليم محددة (إلا من نظام التتبع)
- طلب IBAN أو بيانات بنكية في الشات (وجّه دائماً لرابط النموذج الآمن)
- التظاهر بأنك إنسان — أنت AI ويجب أن تكون صريحاً

💬 أسلوب الكلام:
- ابدأ كل محادثة بـ: "🤖 *QAYDAO AI*: حياك، فريق خدمة العملاء البشري خارج الدوام الآن (٩ ص - ٦ م)، لكن أنا هنا لمساعدتك! 🤍"
- لهجة سعودية ودودة (حياك / أبشر / يسعدنا / تفضل)
- قصير ومباشر (سطرين-ثلاثة كحد أقصى لكل رد)
- استخدم emojis باعتدال: ✅ ⏰ 📦 🚚 🤍

🚨 حوّل فوراً للموظف البشري (private note: "🚨 يحتاج متابعة بشرية") عند:
1. كلمات الغضب: "زعلان / غاضب / مستاء / للأسف / تعبت / بشتكي / محكمة / نصب / كذابين"
2. طلبات الاسترجاع/الإلغاء: "استرجاع / ارجاع المبلغ / الغاء الطلب / كنسل"
3. أي ذكر لـ IBAN أو رقم حساب (وجّه لرابط آمن أولاً)
4. عميل VIP (سيتم وسمه)
5. لو سأل العميل 3 أسئلة ولم تستطع الإجابة على واحدة
6. كلمة صريحة: "أبغى أكلم موظف / تكلم مع إنسان / human / agent"

⏰ في كل رد، اذكر أن الفريق البشري سيتواصل عند بدء الدوام (٩ صباحاً) لمتابعة أي تفاصيل لم تستطع تغطيتها.

🛍️ تخصصك:
- مدد التصنيع والشحن البحري
- سياسات الاسترجاع والإلغاء (التوجيه فقط، لا تنفيذ)
- المنتجات والأسعار العامة
- ساعات العمل وطرق التواصل
- الفروع والمناطق المخدومة
- التركيب والتوصيل

🔐 الخصوصية (مهم — PDPL):
- لا تجمع: IBAN، أرقام بطاقة، صور هوية، أرقام هواتف خاصة
- لو احتاج العميل لإرسال بيانات بنكية، الرد:
  "لحماية بياناتك المالية وفق قوانين حماية البيانات السعودية (PDPL)، يرجى تعبئة النموذج الآمن: https://qaydao.com/refund-form
  سيتم مراجعة طلبك من قسم المالية خلال يوم عمل."
```

### 2.2 إعدادات إضافية

| الحقل | القيمة |
|------|--------|
| **Model** | `gpt-4o-mini` (افتراضي) |
| **Confidence threshold** | `0.7` (إن وُجد) |

### 3.1 الذهاب لقسم Documents

من القائمة الجانبية لـ Captain → **Documents** → **New Document**.

### 3.2 رفع 6 Documents (واحد تلو الآخر)

افتح ملف `CAPTAIN_DOCUMENTS.md` المرفق وانسخ المحتوى من كل قسم بالترتيب.

**الـ 6 Documents المطلوبة:**

| # | الاسم | النوع | الأولوية |
|---|------|------|---------|
| 1 | **شركة قيدداو** | Text | عالية |
| 2 | **ساعات العمل وطرق التواصل** | Text | عالية |
| 3 | **مدة التصنيع والشحن البحري** | Text | حرجة 🔴 |
| 4 | **سياسة الاسترجاع والإلغاء** | Text | حرجة 🔴 |
| 5 | **التوصيل والتركيب** | Text | عالية |
| 6 | **الأسئلة الشائعة (30 سؤال)** | Text | حرجة 🔴 |

## 🎬 الخطوة 4: إنشاء Scenarios (5 سيناريوهات)

Scenarios = playbooks جاهزة للحالات الشائعة. تُوجِّه Captain للرد الأنسب.

من القائمة الجانبية: **Scenarios** → **New Scenario**.

### Scenario 1: تتبع طلب

| الحقل | القيمة |
|------|--------|
| **Title** | `تتبع طلب` |
| **Description** | `العميل يسأل عن حالة طلبه أو ETA الشحن` |
| **Instruction** | اطلب من العميل رقم الطلب (يبدأ بـ 25). إن لم يعطه، اطلب رقم الجوال المسجل. ثم أخبره أن الموظف البشري سيراجع النظام ويرد عند بدء الدوام بحالة الطلب وموعد الشحن المتوقع. لا تخترع تواريخ. |

### Scenario 2: استفسار منتج

| الحقل | القيمة |
|------|--------|
| **Title** | `استفسار منتج` |
| **Description** | `العميل يسأل عن منتج (سعر/مقاس/توفر)` |
| **Instruction** | اشرح ما تعرفه من الـ Documents. لو السؤال عن سعر أو توفر منتج محدد لا تعرفه، وجّه العميل لـ qaydao.com مع رابط مباشر للقسم المعني (أثاث منزلي / مكتبي / كافيهات). اذكر أن الموظف سيتواصل لتزويده بتفاصيل دقيقة. |

### Scenario 3: طلب استرجاع / إلغاء

| الحقل | القيمة |
|------|--------|
| **Title** | `استرجاع أو إلغاء طلب` |
| **Description** | `العميل يريد إلغاء طلبه أو استرجاع مبلغه` |
| **Instruction** | تعاطف مع العميل أولاً ("أعتذر عن أي إزعاج 🤍"). اطلب رقم الطلب. **لا تطلب IBAN أبداً في الشات**. شارك رابط النموذج الآمن: https://qaydao.com/refund-form. أضف private note: "🚨 طلب استرجاع — يحتاج متابعة قسم المالية". |

### Scenario 4: شكوى أو غضب

| الحقل | القيمة |
|------|--------|
| **Title** | `شكوى أو استياء` |
| **Description** | `كشف غضب أو شكوى من العميل` |
| **Instruction** | اعتذر بصدق دون تبرير. لا تجادل. أكّد للعميل أن المشرف العام سيتواصل معه في أول ساعة من بدء الدوام. لا تَعِد بتعويض أو خصم — هذا قرار بشري. أضف private note للمشرف. |

### Scenario 5: استفسار عام / تحية

| الحقل | القيمة |
|------|--------|
| **Title** | `تحية أو استفسار عام` |
| **Description** | `محادثة جديدة بدون سؤال محدد` |
| **Instruction** | رحّب بحرارة، عرّف نفسك ("أنا QAYDAO AI"). اسأل عن كيفية المساعدة. اقترح المواضيع الشائعة: حالة طلب، استفسار منتج، الشحن، الفروع. أكّد أن الموظف البشري سيكون معه بمجرد بدء الدوام. |

| الجمعة | (مغلق) | — |
| السبت | (مغلق) | — |

> ⚠️ **خارج هذه الأوقات → Captain يتولى المحادثة.**

## ✅ الخطوة 6: الاختبار

### 6.1 افتح widget التجريبي

افتح موقع qaydao.com (أو widget URL) **من جوال شخصي** (ليس متصل بـ Chatwoot كموظف).

### 6.2 اختبر 5 سيناريوهات:

| # | اكتب | المتوقع من Captain |
|---|------|---------------------|
| 1 | `السلام عليكم` | ترحيب + تعريف نفسه + اقتراح مواضيع |
| 2 | `كم مدة التوصيل؟` | إجابة من Document مدة التصنيع والشحن |
| 3 | `وين طلبي رقم 250000000؟` | يطلب رقم الطلب + يقول الموظف سيرد بالتفاصيل |
| 4 | `زعلانه منكم، تأخر طلبي شهرين` | اعتذار + private note + وعد بمتابعة المشرف |
| 5 | `أبغى استرجاع المبلغ، حسابي SA45...` | يوقف العميل + يرسل رابط النموذج الآمن |

### 6.3 إذا أي اختبار فشل:

1. ارجع لـ Captain → Logs/Conversations
2. شاهد ما رد به Captain
3. عدّل **Personality** أو أضف **Document** أو **Scenario** جديد
4. أعد الاختبار

## 📊 المتابعة بعد التفعيل (الأسبوع الأول)

### راجع يومياً:

1. **Settings → Reports → Captain** (إن وُجد)
2. **عدد محادثات Captain handled:** يجب أن يكون 10-30/يوم في البداية
3. **عدد escalations:** نسبة طبيعية 20-40%
4. **CSAT للمحادثات المُحلّة بواسطة AI**

### علامات نجاح:

- ✅ Ghost Closures = 0
- ✅ متوسط الرد خارج الدوام < 1 دقيقة
- ✅ زبائن لم يشتكوا من "وين الرد؟"

### علامات تحتاج تعديل:

- ❌ Captain يعطي معلومات خاطئة → أضف/عدّل Documents
- ❌ Captain لا يحوّل عند الغضب → عدّل Personality (قوّي قواعد التحويل)
- ❌ معدل escalation > 70% → غير منطقي، Captain لا يجيب على شيء — راجع الـ Documents

## 🆘 طوارئ: كيف توقف Captain فوراً

لو Captain سبّب مشكلة:

**خيار سريع (لا يحذف شيء):**
1. اذهب لـ Inbox Settings → Captain
2. أوقف ✅ **Enable Captain**
3. احفظ — Captain يتوقف عن الرد في خلال ثوانٍ

**خيار جذري (تعطيل كامل):**

```bash
# عبر SSH على السيرفر
ssh root@69.62.73.5
cd /root/chat-qaydao
# حذف 4 أسطر من .env
sed -i '/CAPTAIN_OPEN_AI_API_KEY/d' .env
sed -i '/CAPTAIN_OPEN_AI_MODEL/d' .env
sed -i '/# Captain AI Agent/d' .env
docker compose restart chatwoot-web chatwoot-sidekiq
```
