# 🤝 QAYDAO Deal Assistant — مساعد الصفقة

نظام داخلي لفريق خدمة عملاء QAYDAO يحوّل طلب "خصم" من العميل إلى **سلة أكبر** عبر المكمّلات، مع نظام عمولات موثّق ومُدقَّق للموظفين.

> **الفلسفة:** لا تُنزّل السعر — زِد القيمة.

## ✨ الميزات

### للموظف (`/deal`)
- 🔐 **هوية تلقائية من Chatwoot** — الموظف المسجّل دخوله في `chat.qaydao.com` يُعرَف فوراً (تحقق server-side من كوكي الجلسة)، مع اختيار يدوي كاحتياط (يُوسم "غير موثّق")
- 🔍 بحث حي في كتالوج 9,700+ منتج (PostgreSQL، اتصال **قراءة-فقط**)
- 💸 أزرار خصم (0/3/5/7/10٪) مع سعر بعد الخصم فوراً
- 🧲 مكمّلات: بحث + chips فئات + مقترحات تلقائية، وكل مكمّل يعرض "+X لعمولتك"
- 💰 **عدّاد عمولة حي متحرك** + شريط شهري (عمولتك، ترتيبك، صفقاتك)
- 🗣️ سكربت إقناع ذكي متغيّر + نسخ العرض + إرسال واتساب
- 🎟️ كود كوبون الموظف (يُتذكّر، يظهر في العرض، يُحفظ مع الصفقة)
- 🎉 احتفال عند تسجيل الصفقة

### للموظف (`/deal/me`) — متابعتي
لوحة شخصية: عمولة الشهر (معتمد/معلّق) · الترتيب · متوسط السلة · شارات إنجاز · صدارة الشهر · آخر الصفقات · رسالة تحفيزية ذكية.

### للإدارة (`/deal/report`) — مفتاح إداري منفصل
KPIs · ملخص الموظفين · **تدقيق كل طلب**: مطابقة رقم الطلب مع سلة ثم **اعتماد/رفض/تعليق** · شارة 🔐/✍️ لمصدر الهوية.

## 🛡️ الأمان (مبادئ التصميم)

| المبدأ | التنفيذ |
|---|---|
| العمولة لا تُزوَّر | **تُحسب في الخادم فقط** — قيم العميل تُتجاهل |
| الهوية لا تُنتحل | تحقق server-side من جلسة Chatwoot (`/api/v1/profile`) |
| رقم الطلب لا يُسجَّل مرتين | فهرس فريد جزئي (`WHERE status <> 'rejected'`) — الرفض يحرّر الرقم |
| كل صفقة تُدقَّق | `status: pending → approved/rejected` بمفتاح إداري منفصل |
| الكتالوج لا يُمس | اتصال القراءة `default_transaction_read_only=on` على مستوى Postgres |
| البيانات المالية محمية | نسخ يومي `pg_dump` محلي (14 يوم) + رفع R2 |

## 🏗️ المعمارية

```
nginx (chat.qaydao.com/deal) → Node/Express (127.0.0.1:3611, pm2)
├── db.js          → PostgreSQL qaydao_master (قراءة-فقط: master_products)
├── recordsDb.js   → PostgreSQL (كتابة: deal_records فقط)
└── Chatwoot API   → هوية الموظفين (profile + agents)
public/  index.html (الأداة) · me.html (متابعتي) · report.html (الإدارة)
```

## 🔌 API

| Endpoint | الوصف |
|---|---|
| `GET /deal/api/search?q=` | بحث الكتالوج |
| `GET /deal/api/complements?category=` | مقترحات مكمّلة |
| `GET /deal/api/whoami` | هوية Chatwoot الموثّقة |
| `GET /deal/api/agents` | قائمة الموظفين (fallback) |
| `POST /deal/api/record` | تسجيل صفقة (إعادة حساب خادمية) |
| `GET /deal/api/me?month=` | لوحة الموظف |
| `GET /deal/api/report?month=` 🔑 | بيانات التقرير (إداري) |
| `POST /deal/api/review` 🔑 | اعتماد/رفض (إداري) |

## ⚙️ التشغيل

```bash
cp .env.example .env   # واملأ القيم
npm install
pm2 start server.js --name qaydao-deal
```
